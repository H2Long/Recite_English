# 背单词软件改进建议

## 一、功能增强建议

### 1. 学习模式优化

| 功能 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| 艾宾浩斯遗忘曲线 | 目前认识3次即掌握 | 引入间隔重复算法，根据遗忘曲线自动安排复习时间 | ⭐⭐⭐ 高 |
| 单词发音 | 无 | 添加 TTS 文字转语音，播放单词和例句发音 | ⭐⭐ 中 |
| 拼写练习 | 仅有选择题 | 新增听写/拼写模式，锻炼拼写能力 | ⭐⭐ 中 |
| 生词本/收藏夹 | 无 | 支持标记重点单词，方便后续复习 | ⭐⭐⭐ 高 |

### 2. 测试模式增强

| 功能 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| 测试题数设置 | 固定最多10题 | 支持自定义题数（5/10/20/全部） | ⭐⭐ 中 |
| 错题重做 | 无 | 自动收集错题，生成错题专项训练 | ⭐⭐⭐ 高 |
| 测试类型 | 仅选择题 | 新增：听音选词、拼写补全、判断题 | ⭐⭐ 中 |
| 计时模式 | 无 | 添加计时器，增加学习紧迫感 | ⭐ 低 |

### 3. 背单词模式改进

| 功能 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| 卡片样式 | 简单翻转 | 添加 3D 翻转动画、更多主题样式 | ⭐ 低 |
| 学习范围 | 全部/未掌握 | 支持按首字母筛选、按日期筛选 | ⭐⭐ 中 |
| 每日目标 | 无 | 设置每日学习目标（如每天背20个），显示完成进度 | ⭐⭐⭐ 高 |

## 二、数据管理改进

### 1. 单词数据管理

```
现状：
- 单词来源：words.txt 或内置10个默认单词
- 格式：word|phonetic|definition|example

建议改进：
- 支持 Excel/CSV 导入导出
- 支持在线词典 API 查询（如有道、金山词霸）
- 添加单词搜索功能
- 支持单词分类/标签管理
```

### 2. 学习进度管理

| 功能 | 现状 | 建议 |
|------|------|------|
| 进度统计 | 仅显示掌握/学习中数量 | 添加：今日学习数、连续学习天数、学习热力图 |
| 数据导出 | 仅保存 progress.txt | 支持导出为 Excel、JSON 格式 |
| 多用户支持 | 所有用户共用同一进度 | 支持切换用户/ profiles，数据隔离 |
| 云同步 | 无 | 集成云端同步（可选），防止数据丢失 |

## 四、技术架构优化

### 1. 代码结构

```
当前问题：
- 大量全局变量分散管理
- UI 组件和业务逻辑耦合较紧
- fonts.c 和 raylib_word_ui.c 有重复的字体处理逻辑

建议：
- 引入状态管理模式，集中管理应用状态
- 抽离 Renderer 模块，UI 与渲染解耦
- 统一字体处理逻辑，合并重复代码
- 分离数据层（Model）、业务层（Service）、表现层（UI）
```

### 2. 性能优化

| 方向 | 建议 |
|------|------|
| 字体加载 | 字体加载较慢，考虑异步加载或延迟加载 |
| 渲染优化 | 复杂页面考虑脏矩形渲染，避免全量重绘 |
| 数据存储 | 大单词库考虑 SQLite 替代纯文本存储 |
| 内存管理 | 检查潜在的内存泄漏，特别是动态分配的字符串 |

### 3. 可扩展性

| 方向 | 建议 |
|------|------|
| 插件系统 | 设计插件接口，支持扩展题库、卡片样式等 |
| 配置文件 | 将硬编码参数提取到 config.json |
| 国际化 | 预留 i18n 接口，便于多语言支持 |
| 日志系统 | 添加日志记录，便于问题排查 |

## 五、推荐优先级

### 🔴 高优先级（建议优先实现）

1. **间隔重复算法** - 科学的复习计划是背单词软件的核心价值
2. **每日学习目标** - 提升用户粘性和学习动力
3. **深色模式** - 用户呼声较高，实现相对简单
4. **错题本功能** - 针对性复习，提高学习效率
5. **搜索功能** - 单词多了之后必须的功能

### 🟡 中优先级（提升体验）

6. 单词发音（TTS）
7. 快捷键支持
8. 每日/每周学习统计
9. 单词分类/标签
10. 窗口大小可调

### 🟢 低优先级（锦上添花）

11. 3D 卡片动画
12. 音效反馈
13. 新手引导
14. 云同步
15. 插件系统
