# 基于 C 语言的跨平台智能英语单词记忆系统 —— 课程设计报告

---

## 摘要

本文阐述了一款面向英语学习者的桌面级单词记忆软件的设计与实现。系统采用 **C11 标准**与 **raylib 多媒体图形库**构建，通过**即时模式图形用户界面（Immediate Mode GUI）**范式，在 Linux、Windows 及 macOS 平台实现一致的交互体验。系统核心包含用户账户管理、单词库维护、多模式智能复习、自适应学习计划及跨语言文字渲染五大子系统。设计上遵循**全局单例状态管理**与**函数指针驱动导航**的架构模式，以纯文本管道分隔文件实现轻量级数据持久化，不依赖外部数据库引擎，真正做到零配置、开箱即用。

**关键词：** 即时模式 GUI；间隔重复；C11；跨平台图形应用；自适应学习系统

## 一、系统总体架构

### 1.1 架构概览

系统采用 **分层单向依赖** 架构，自底向上分为数据层、业务逻辑层、UI 基础组件层与页面展示层。各层之间通过全局单例状态对象 `AppState` 进行数据共享，避免跨层指针传递。

```

│                     页面展示层                              │
│  主页 │ 浏览学习 │ 卡片背单词 │ 选词背单词 │ 测试 │ 搜索   │
│       学习计划 │ 设置 │ 账号管理 │ 词库管理                 │
├─────────────────────────────────────────────────────────────┤
│                   UI 基础组件层                             │
│  按钮/复选框/文本框 │ 滚动视图 │ 搜索栏 │ 单词卡片/抽认卡  │
│            多项选择题 │ 跨文字渲染引擎                      │
├─────────────────────────────────────────────────────────────┤
│                    业务逻辑层                               │
│  账户系统 │ 单词库引擎 │ 学习计划引擎 │ 搜索匹配 │ 进度追踪 │
├─────────────────────────────────────────────────────────────┤
│                      数据层                                 │
│  accounts.txt  words.txt  progress.txt  plans.txt           │
├─────────────────────────────────────────────────────────────┤
│                     核心框架                                │
│     全局状态管理器(AppState) │ 树形菜单导航 │ 主循环调度    │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心控制模块

### 2.1 全局状态管理器（core/app_state.c）

作为整个系统的**数据中枢**，`AppState` 采用单例模式将所有可变运行时状态统一管理。各业务模块通过依赖注入方式持有指向其子状态的指针，实现低耦合数据共享。

```
AppState (全局单例)
  │
  ├── UIStyle style ────────── 主题配色 + 字体 + 字号 + 间距
  ├── UIState uiState ──────── 每帧鼠标位置/按键状态、hotItem/activeItem/focusItem
  ├── bool isDarkMode ──────── 深色/浅色主题标记
  ├── MENU* rootMenu ───────── 导航树根节点
  ├── MENU* currentMenu ────── 当前活动页面节点
  ├── MenuStack menuStack ──── 导航历史栈（最大深度10）
  │
  ├── LearnState learn ──────── 浏览模式: 当前词索引、仅显示未掌握、滚动偏移
  ├── ReviewState review ────── 抽认卡: 待复习列表、当前索引、会话统计
  ├── TestState test ────────── 测试: 题目数、当前题号、正确数、是否完成
  ├── SelectWordState select ── 选词模式: 题目列表、当前题号、正确数
  ├── SearchState search ────── 搜索: 查询字符串、结果列表、滚动偏移
  │
  ├── AccountState account ──── 账户状态（由 account.c 操作）
  ├── PlanState plan ────────── 计划状态（由 plan.c 操作）
  └── bool loginRequired ────── 是否要求登录
```

**设计优势：** 传统的面向对象做法会为每个对话框分配独立状态对象，导致跨页面状态共享困难。采用全局单例后，从"浏览学习"页面切换到"抽认卡"页面时，无需序列化/反序列化状态，只需切换导航指针即可。

### 2.2 树形菜单导航（core/tree_menu.c）

系统采用 **多路树数据结构** 组织页面导航。每个菜单节点持有一个 `show` 函数指针，运行时通过 `CURRENT_MENU->show()` 多态调用对应页面的渲染函数。

**菜单树结构：**

```
根节点 (Root)
  ├── 学单词 ─────────────── MenuLearn_Show
  ├── 背单词 ─────────────── MenuReviewRoot_Show
  │     ├── 卡片背单词 ───── MenuCardReview_Show
  │     ├── 选词背单词 ───── MenuSelectWord_Show
  │     └── 测试 ─────────── MenuTest_Show
  ├── 查找单词 ───────────── MenuSearch_Show
  ├── 学习计划 ───────────── MenuPlanRoot_Show
  │     ├── 计划管理 ─────── MenuPlanManager_Show
  │     └── 学习进度 ─────── MenuProgress_Show
  ├── 设置 ───────────────── MenuSettings_Show
  ├── 账号管理 ───────────── MenuAccount_Show
  └── 词库管理 ───────────── MenuWordManager_Show
```

**导航栈操作流程：**

```
用户点击 "卡片背单词"
  │
  ├─ StackPush(stack, currentMenu)    // 将"主页"压入栈
  ├─ currentMenu = 卡片背单词节点      // 切换页面
  │
  └─ ...用户操作后点击 "返回"...
       │
       ├─ StackPop(stack)              // 弹出栈顶
       └─ currentMenu = 栈顶页面       // 恢复"主页"
```

**核心数据结构：**

```c
typedef struct menu {
    void (*show)(void);          // 页面渲染函数指针 —— 实现多态导航
    struct menu* child[8];       // 子节点数组，最大出度 8
    struct menu* parent;         // 父节点指针，支持回溯
    int childindex;              // 已挂载子节点计数
} MENU;

typedef struct {
    MENU* menuStack[10];         // 最大深度 10 层的导航栈
    int Stacktop;                // -1 表示空栈
} MenuStack;
```

## 三、业务逻辑模块

### 3.1 用户账户系统（modules/account.c）

实现多用户隔离的账户管理，支持注册、登录、注销与销户操作。密码采用 **djb2 哈希算法** 存储摘要，避免明文泄露风险。

**初始化流程：**

```
程序启动
  │
  ├─ accounts.txt 存在？
  │     ├─ 是 → 逐行解析管道分隔字段，加载至 User 数组
  │     └─ 否 → 创建默认管理员账户 admin/admin，写入文件
  │
  └─ 初始化完成
```

**登录流程：**

```
用户输入用户名 + 密码
  │
  ├─ 遍历 User 数组比对用户名
  │     └─ 未找到 → 返回 "用户不存在"
  │
  ├─ djb2(输入密码) 与存储的 passwordHash 比较
  │     └─ 不匹配 → 返回 "密码错误"
  │
  └─ 匹配成功
       ├─ currentUserIndex = 该用户索引
       ├─ 进度文件切换: progress.txt → progress_<username>.txt
       ├─ 计划文件切换: plans.txt → plans_<username>.txt
       └─ 更新 lastLoginTime，持久化至 accounts.txt
```

**多用户文件隔离：**

```
未登录状态:            progress.txt          plans.txt
                          │                     │
Alice 登录 ──────────────→ progress_alice.txt   plans_alice.txt
                          │                     │
注销 ───────────────────→ progress.txt          plans.txt
                          │                     │
Bob 登录 ────────────────→ progress_bob.txt     plans_bob.txt
```

**密码安全设计：** djb2 哈希算法虽非密码学安全级别（如 bcrypt），但相比明文存储，有效防止了直接查看配置文件导致的密码泄露。其轻量级特性避免了外部加密库依赖，适合本地桌面应用场景。

### 3.2 单词库引擎（modules/words.c）

实现单词数据的 CRUD 操作、无序索引至有序进度数组的映射、以及基于通配符的模糊搜索算法。

**数据映射流程：**

```
words.txt (磁盘)
  │  word|phonetic|definition|example|translation
  │
  ▼ load_words_from_file()  逐行解析管道分隔
  │
g_wordLibrary[] (动态数组, 按文件顺序)
  │  WordEntry { char* word, *phonetic, *definition, *example, *translation }
  │
  ▼ init_words()  浅拷贝指针 + 零初始化进度字段
  │
g_words[300] (固定数组, 携带进度)
  │  WordWithProgress { WordEntry entry; WordProgress progress; }
  │  progress { knownCount, unknownCount, lastReview, mastered }
  │
  ▼ load_progress() / save_progress()
  │
progress.txt (磁盘)
  │  word|knownCount|unknownCount|lastReview
```

**通配符搜索算法：**

```
用户输入搜索模式 (如 "ab*")
  │
  ├─ 模式含通配符 (^ $ * ?) ？
  │     ├─ 是 → search_words_by_regex()  自实现迷你正则引擎
  │     │       支持语法:
  │     │         * → 零个或多个字符
  │     │         ? → 恰好一个字符
  │     │         ^ → 锚定开头
  │     │         $ → 锚定结尾
  │     │
  │     └─ 匹配数 > 0 → 返回通配符匹配结果
  │            匹配数 = 0 → 回退到子串匹配
  │
  └─ 模式无通配符 → search_words_simple()
        └─ 不区分大小写子串匹配，返回结果
```

**检索示例：**

| 输入模式 | 匹配逻辑 | 示例匹配 |
|----------|---------|---------|
| `ab*` | 以 "ab" 开头 | abandon, abstract |
| `*tion` | 以 "tion" 结尾 | education, information |
| `h?t` | "h" + 任一字符 + "t" | hat, hit, hot |
| `^test` | 精确开头为 "test" | test, testing |

### 3.3 学习计划引擎（modules/plan.c）

实现自适应学习计划管理，支持计划的创建、激活与**跨日期自动推进**。

**跨日自动推进流程：**

```
用户每完成一个单词学习
  │
  └─ Plan_AddStudiedToday(1)
       │
       └─ Plan_CheckNewDay()
            │
            ├─ 获取当前日期 yyyymmdd
            │
            ├─ 与 lastStudyDate 比较
            │     ├─ 同一天 → studiedToday += 1
            │     │
            │     └─ 跨天 → currentDay++
            │              studiedToday = 1
            │              lastStudyDate = today
            │
            └─ 更新 plans.txt
                 │
                 ├─ studiedToday >= dailyWordCount → "今日目标已达成!"
                 └─ studiedToday < dailyWordCount  → "今日剩余: N 个单词"
```

**四大预设学习方案：**

| 计划名称 | 每日单词量 | 总天数 | 累计覆盖 |
|---------|-----------|-------|---------|
| 一周入门计划 | 10 词/天 | 7 天 | 70 词 |
| 半月巩固计划 | 15 词/天 | 15 天 | 225 词 |
| 三十天进阶计划 | 20 词/天 | 30 天 | 600 词 |
| 六十天冲刺计划 | 30 词/天 | 60 天 | 1800 词 |

### 3.4 复习策略与进度追踪

系统采用**类间隔重复（Spaced Repetition）** 策略跟踪每个单词的学习状态。

```
RefreshReviewList() 筛选待复习单词
  │
  └─ 遍历 g_words[]，收集未掌握的单词 → reviewWords[]
       │
       └─ reviewWords 为空 → "恭喜！所有单词已掌握!"
       └─ reviewWords 非空 → 逐个展示抽认卡
            │
            ├─ 卡片正面: 显示英文单词
            │
            ├─ 用户点击翻转 → 卡片背面: 显示中文释义 + 音标
            │
            ├─ 用户判断:
            │     ├─ "认识" → knownCount++
            │     └─ "不认识" → unknownCount++
            │
            ├─ 更新 lastReview
            │
            ├─ knownCount >= 3 (MASTERED_THRESHOLD) ?
            │     ├─ 是 → mastered = true，移出复习池
            │     └─ 否 → 保留，下次继续复习
            │
            └─ 继续下一个单词

掌握度判别 (颜色标记):
  掌握率 = knownCount / (knownCount + unknownCount)
    ├─ ≥ 80% → 绿色 (已掌握)
    ├─ 50% ~ 79% → 橙色 (学习中)
    └─ < 50% → 红色 (薄弱)
```

## 四、UI 基础组件层

### 4.1 即时模式 GUI 系统（ui/raylib_word_ui.c）

本系统采用**即时模式（Immediate Mode）** GUI 范式，区别于传统的保留模式（Retained Mode）。每个 UI 控件以函数的形式存在，在同一调用中同时完成输入处理与视觉渲染。

**两种范式对比：**

```
保留模式 (Qt/GTK):
  CreateWidget() → SetProperties() → 控件自持状态 → 事件循环驱动更新

即时模式 (本系统):
  每帧调用 UIButton() → 检测鼠标位置 → 更新 hotItem/activeItem → 绘制 → 返回点击状态
```

**UI 控件热点状态机：**

```
每帧开始: hotItem = 0, activeItem = 0
  │
  ├─ 鼠标进入控件区域 → hotItem = 该控件ID     (热态)
  │     ├─ 鼠标按下 → activeItem = 该控件ID    (激活态)
  │     │     ├─ 鼠标松开 → 触发点击回调       (完成一次交互)
  │     │     └─ 鼠标保持按下 → 保持激活态     (拖拽中)
  │     └─ 鼠标离开 → hotItem = 0             (离开)
  │
  └─ 下一帧重新开始
```

**主题系统：**

```
浅色主题 ──────────── ToggleDarkMode() ────────────→ 深色主题
  背景: #F5F5F5                                         背景: #1E1E2E
  卡片: #FFFFFF                                         卡片: #2D2D3F
  主文字: #333333                                       主文字: #E0E0E0
  主色调: #4A90D9                                       主色调: #6CB4EE
  成功色: #67C23A                                       成功色: #7ED321
  危险色: #F56C6C                                       危险色: #FF6B6B
  ←──────────────── ToggleDarkMode() ─────────────────
```

### 4.2 多语言文字渲染引擎（modules/fonts.c）

针对中英混排场景，实现了**基于 Unicode 码位的动态字体切换**渲染引擎。

**逐字符渲染流程：**

```
输入文本: "abandon /əˈbændən/ 放弃"
  │
  └─ 逐字符遍历:
       │
       ├─ 'a','b','a','n','d','o','n' ...
       │     └─ U+0000~U+007F (ASCII) → 英文字体 (DejaVuSans.ttf)
       │
       ├─ 'ə','æ' ...
       │     └─ U+0080~U+024F (IPA音标) → 拉丁字体 (DejaVuSans + IPA扩展码位)
       │
       ├─ '放','弃'
       │     └─ U+4E00~U+9FFF (中文) → 合并字体 (NotoSansCJK + 默认字体回退)
       │
       └─ 每字符累加宽度，推进绘制坐标 → 完成
```

**字体后备链：**

```
英文字体加载:
  首选 data/fonts/DejaVuSans.ttf
    ↓ 失败
  Windows: C:/Windows/Fonts/arial.ttf
    ↓ 失败
  macOS: /System/Library/Fonts/Helvetica.ttc
    ↓ 失败
  Linux: /usr/share/fonts/truetype/dejavu/DejaVuSans.ttf
    ↓ 失败
  最终回退: GetFontDefault()

中文字体加载:
  首选 data/fonts/NotoSansCJK.otf
    ↓ 失败
  平台字体回退链...
```

### 4.3 抽认卡翻转动画

卡片背单词功能采用**三维仿射变换**模拟卡片翻转效果：

```
每帧调用 UIFlashCard():
  │
  ├─ 计算翻转进度 t (Lerp 指数平滑插值)
  │
  ├─ t < 0.5 (卡片正面):
  │     scaleX = cos(t * PI)    // 从 1 平滑收缩到 0
  │     显示: 英文单词
  │
  ├─ t >= 0.5 (卡片背面):
  │     scaleX = cos(t * PI)    // 从 0 翻转到 -1
  │     显示: 中文释义 + 音标
  │
  ├─ DrawTextPro() 以卡片中心为锚点缩放 X 轴 → 产生 3D 翻转视觉
  │
  └─ 底部: "认识" / "不认识" 按钮
       └─ 点击后 t 重置，载入下一单词
```

### 5.1 页面职能矩阵

## 六、数据持久化方案

### 6.1 文件存储设计

系统采用**管道分隔纯文本格式**作为存储方案，避免引入 SQLite 等外部依赖，确保可读性、可编辑性和零配置部署。

| 文件 | 字段结构 | 编码 |
|------|---------|------|
| `words.txt` | `word\|phonetic\|definition\|example\|translation` | UTF-8 |
| `accounts.txt` | `username\|passwordHash\|createdTime\|lastLoginTime\|correct\|total` | UTF-8 |
| `progress.txt` | `word\|knownCount\|unknownCount\|lastReview` | UTF-8 |
| `plans.txt` | `name\|dailyWordCount\|totalDays\|currentDay\|createdAt\|lastStudyDate\|studiedToday\|isActive` | UTF-8 |

**示例数据 (words.txt):**

```
abandon|əˈbændən|放弃、遗弃|He abandoned his car in the snow.|他把汽车丢在雪地里。
abstract|ˈæbstrækt|抽象的、摘要|The idea is too abstract to understand.|这个想法太抽象了，难以理解。
```

### 6.2 多用户数据隔离方案

```
未登录 ───────── progress.txt ───────── plans.txt
                      │                      │
Alice登录 ──→ progress_alice.txt ──→ plans_alice.txt
                      │                      │
注销 ──────→ progress.txt ───────── plans.txt
                      │                      │
Bob登录 ────→ progress_bob.txt ────→ plans_bob.txt
```

登录时通过 `Account_GetProgressPath(buf)` / `Account_GetPlanPath(buf)` 构建用户名专属路径，然后调用 `set_progress_file_path()` / `Plan_SetFilePath()` 重定向所有后续读写操作。

## 八、关键技术决策与创新点

### 8.1 设计决策权衡

| 决策维度 | 选择方案 | 选择理由 | 代价 |
|---------|---------|---------|------|
| GUI 范式 | 即时模式 | 无对象生命周期管理；代码量少 60%；状态完全显式 | 每帧重绘全部控件，CPU 消耗略高 |
| 数据存储 | 纯文本管道分隔 | 零依赖；记事本可编辑；版本控制友好 | 无并发支持；大数据量下性能差 |
| 编程语言 | C11 | 极致性能；极小二进制体积；与 raylib C API 无缝对接 | 无 STL，手工管理字符串与数组 |
| 密码安全 | djb2 哈希 | 无外部加密库依赖；比明文安全 | 非密码学安全，不适于网络场景 |
| 字体渲染 | 自研 DrawTextAuto | 解决中英混排；无额外库依赖 | 维护成本；需各平台适配 |

### 8.2 创新亮点

**1. 即时模式 GUI + 主题系统：** 在没有 GTK/Qt 等重型框架的情况下，从零实现了完整可用的图形界面组件库，包含按钮、文本框、复选框、滚动视图、搜索栏、单词卡片、抽认卡、多项选择题等丰富控件，支持深浅双主题一键切换。

**2. Unicode 级字体切换：** 针对 raylib 原生仅支持单一字体渲染的限制，设计了逐字符码位检测 + 动态字体切换算法，实现了 ASCII 英文、IPA 国际音标、CJK 中文字符在同一段落内的和谐共存。

**3. 多用户数据隔离：** 通过文件路径重定向（而非数据库字段）实现轻量级多用户隔离，每个用户的学习进度和计划完全独立，且切换零开销。

**4. 自适应学习计划：** 计划引擎自动感知日期跨越，无需用户手动签到即可实现 `currentDay` 推进与每日计数器重置，真正做到无感追踪。

## 九、总结

本系统以 C11 语言配合 raylib 图形库，实现了一款功能完备的跨平台英语单词记忆桌面软件。系统涵盖用户管理、单词库维护、四种学习模式、自适应计划管理等核心功能模块，通过即时模式 GUI 范式与 Unicode 级字体切换引擎，在轻量级的技术栈上实现了良好的用户体验。管道分隔纯文本文件作为数据持久化方案，确保了系统的零配置部署与高可维护性。项目通过 GitHub Actions 实现 Linux AppImage 与 Windows zip 双平台自动构建发布，具备完整的工程化交付能力。
